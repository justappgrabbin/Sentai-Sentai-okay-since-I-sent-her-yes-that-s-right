# Synthia Morph OS v2

Corrected assembly for the Morph OS direction.

This is not a five-state app shell. It is a recursive field-address operating system with a smart browser, user-relative dashboard, 384 materialized spectrum nodes, universal ingestion starter, MCP continuity packets, and a problem-oriented orchestrator.

## Run

```bash
npm install
npm run start
```

Open the Vite URL shown in the terminal, usually `http://localhost:5173`.

The local MCP continuity server runs at `http://localhost:3030`.

## What is included

- Dashboard with embedded admin panel for admin role
- Five OS node pages: Movement, Memory, Relation, Field, Action
- Movement smart browser iframe as an allowed browser viewport
- Universal file/conversation ingestion into field-address capsules
- 384 user-local spectrum nodes, with 22T+ virtual address-space note
- Message-passing morph bus
- MCP continuity packet generator
- Problem-oriented orchestrator starter
- Supabase schema text and localStorage working memory
- Neo4j canon adapter stub
- Heuristic equivalence / repeatability starter

## Build rule

Morphing happens through message passing across existing address relations. The system does not rewrite its architecture at runtime. Expansion means adding nodes, address faces, capsules, and relations.
