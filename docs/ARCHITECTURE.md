# Synthia Morph OS v2 Architecture

## Core law

Everything is a field. Every field has many address-faces. Morphing is translation across those faces while preserving identity. Message passing activates translation.

## Address scale

The whole system can support at least 22T virtual addresses, but a user-local UI materializes 384 primary spectrum nodes: 64 hexagrams × 6 lines. Deeper address faces are lazy/virtual and activated only as needed so the interface does not crush a phone or browser.

## Visible OS

- Dashboard: customizable home surface; admin controls are embedded here for admin users.
- Movement: smart browser, live motion, uploads entering motion, active session events.
- Memory: raw uploads, observed capsules, Supabase staging.
- Relation: people, agents, MCP packets, collaborators, reference/friend graph.
- Field: 384 materialized nodes, color/shape/number/symbol/code/sound address faces.
- Action: deployments, automations, tests, approvals.

## Iframes

Only two iframe roles are allowed:

1. Smart browser viewport under Movement.
2. Sandbox preview for unfruited capsules/components.

Everything else is native React/JS state and message-passing behavior.

## Memory and canon

Supabase/local storage is working memory. Neo4j is the later canonical graph. Relations mature through raw → observed → hypothesized → tested → repeated → canonical-ready → canonical.
